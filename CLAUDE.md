## Git 運用ルール

### 基本方針

**コードを変更するたびに GitHub へプッシュする。** ローカルだけに変更を溜めない。

### コミット手順

1. 変更をステージング: `git add <ファイル名>` (機密ファイルは除く)
2. コミット作成（変更の「理由」を書く）:
   ```bash
   git commit -m "メッセージ"
   ```
3. **即座に GitHub へプッシュ**:
   ```bash
   git push origin <ブランチ名>
   ```

### ブランチ戦略

- `main` — 本番相当。直接プッシュ禁止。
- `feature/<機能名>` — 新機能開発。
- `fix/<内容>` — バグ修正。

機能追加・修正はブランチを切り、PR でマージする。

### コミットメッセージ規約

```
<type>: <要約（日本語 or 英語）>
```

| type | 用途 |
|------|------|
| feat | 新機能 |
| fix | バグ修正 |
| refactor | リファクタリング |
| test | テスト追加・修正 |
| docs | ドキュメント変更 |
| chore | ビルド・設定変更 |

例: `feat: 物件一覧ページに絞り込みフィルターを追加`

### Claude Code がコードを変更した場合

Claude Code がコードを変更するたびに、以下を実行する:

```bash
git add <変更したファイル>
git commit -m "<type>: <変更内容の要約>"
git push origin <現在のブランチ名>
```

`git push` は省略しない。変更のたびに必ず実行すること。
